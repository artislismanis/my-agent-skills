# C4 Diagram Quick Reference

**Standard**: C4 Model (Simon Brown) | **Levels**: Context (L1), Container (L2), Component (L3)

## Shape → Element Mapping

| C4 Concept | Shape | Background |
|------------|-------|------------|
| Person / User | `rectangle` + `roundness: {type:3}` | `#d3f9d8` |
| System (in scope) | `rectangle` | `#dbe4ff` |
| External System | `rectangle` | `#f1f3f5` |
| Database / Queue | `rectangle` | `#fff3bf` |
| Component / Interface | `rectangle` | `#dbe4ff` |
| System / Container Boundary | `frame` | `transparent` |

## Label Conventions

- **L1 (Context)**: `"Name\n[Person]"` or `"Name\n[Software System]"`
- **L2 (Container)**: `"Name\n[Container: Technology]\nDescription"`
- **L3 (Component)**: `"Name\n[Component: Technology]\nResponsibility"`
- Arrow labels: describe the interaction (protocol, verb phrase)

## Layout

- In-scope system at centre; users top/left; external systems right/bottom
- Wrap in-scope elements in a boundary frame (40px padding)
- L2: databases at bottom of boundary; external systems outside
- L3: group by layer (controllers → services → repositories)

## Rules

- Always include `[Type]` tag as the second label line
- Use frames for ALL boundary types
- Limit: 5–10 elements (L1), 5–15 elements (L2)
- Do not mix C4 levels in a single diagram
